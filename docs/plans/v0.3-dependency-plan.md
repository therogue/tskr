## Claude-Recommended ordering

### Phase 1 — independent quick wins, parallelizable

- #68 — already in PR #74; finish review and merge.
- #79 — `SECURITY.md`, docs only.
- #71 — calendar full-width fix.
- #72 — completed tasks stay visible.
- #69 + #70 — selection-state cluster, single PR or stacked.

### Phase 2 — backlog cluster (sequential)

1. #67 overdue → backlog. Establishes backend rule: clear `scheduled_date` with skip rules (completed=false, is_template=false, parent_task_id=null). Frontend app-load trigger.
2. #66 bulk-move-backlog. Reuses skip-rule helper from #67; adds `POST /tasks/move-to-backlog` and the two UI entry points (move-all / move-selected).

Reason: doing #67 first lets #66 reuse the eligibility filter rather than inventing it twice.

### Phase 3 — recurrence and observability, parallelizable

- #73 recurrence-on-update. Backend mirrors create-time template/instance behavior.
- #63 Langfuse. Backend wrap of Anthropic client; optional via env.

### Phase 4 — UI revamp (#86 umbrella)

Recommended order inside the umbrella:

1. **#87 fixed-width tabs.** Layout baseline. Without this, widget area sizing is underspecified.
2. **#88 productive widgets.** Adds the right-panel widgets that #89 will hide.
3. **#89 chat-covers-widgets.** Pure interaction over the layout from #87+#88.
4. **#91 task modal view.** Independent of widget chain; can run in parallel from start of Phase 4.
5. **#92 task cards in chat.** Likely shares card/modal components with #91; do after.
6. **#90 suggestive follow-ups.** Backend+ai; can start in parallel from Phase 3 or Phase 4.
